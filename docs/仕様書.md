# KenSapo システム仕様書

## 1. 基本情報

- システム名: KenSapo（ケンサポ）
- 製品区分: 建設業向け現場スケジュール管理システム
- 本番URL: https://kensapo.vercel.app
- 対象: 建設会社の管理者・現場監督・作業員・事務担当者
- 対応端末: iPhone、Android、PCのモダンブラウザ

## 2. システム概要

KenSapoは次の業務を会社単位で管理します。

- 現場情報
- 作業員
- 作業予定と開始・終了実績
- 現場写真
- 日報
- 工事進行チェック
- Web Push通知

## 3. 技術構成

- フロントエンド: Next.js 16、React、TypeScript、Tailwind CSS
- 認証・データベース・写真保存: Supabase
- ホスティング: Vercel
- 通知: Service Worker、Web Push、VAPID

## 4. 主な画面

- `/login`: ログイン・新規登録
- `/dashboard`: ホーム・今日の概要
- `/schedule`: 作業予定・開始・終了
- `/schedule/new`: 予定登録
- `/sites`: 現場一覧
- `/sites/new`: 現場登録
- `/sites/[id]`: 現場詳細
- `/sites/[id]/photos`: 現場写真
- `/sites/[id]/reports`: 日報一覧
- `/sites/[id]/reports/new`: 日報登録
- `/sites/[id]/progress`: 工事進行
- `/workers`: 作業員一覧
- `/workers/new`: 作業員登録
- `/settings`: 通知・認証・アカウント設定

旧URLの一部は互換性のため新しい画面へ転送します。

## 5. 機能仕様

### 認証

- メールアドレスとパスワードによるログイン
- 会社と管理者アカウントの新規登録
- 対応端末での顔認証・指紋認証
- 未ログイン時のログイン画面への転送

### 現場

- 現場名、住所、担当者、期間、状態、メモの登録・更新
- 状態: 未着手、進行中、完了
- 新規現場登録時の標準工事進行項目作成

### 作業員・予定

- 作業員の名前、電話番号、職種、在籍状態の管理
- 日付、現場、担当作業員、開始・終了予定、作業内容の管理
- 当日の作業開始・終了実績の記録

### 写真

- Supabase Storageへの写真保存
- タイトル、コメント、撮影日の記録
- 会社・現場単位のアクセス制御

### 日報

- 日付、天気、人数、作業内容、材料、問題点、翌日の予定の記録
- 現場単位の日報一覧

### 工事進行

- カテゴリ・区分ごとのチェックリスト
- 完了・未完了の切り替え
- 現場単位の進捗率表示

### 通知

- 通知設定をオンにした端末へのWeb Push通知
- 対象: 予定、作業開始・終了、現場、日報、写真、工事進行の更新
- 同じ会社の他ユーザーへ通知
- 通知タップ時に関連画面を表示

## 6. データベース

主要テーブル:

- `companies`: 会社
- `profiles`: ユーザー情報
- `projects`: 現場
- `workers`: 作業員
- `schedules`: 予定と実績
- `site_photos`: 写真情報
- `daily_reports`: 日報
- `project_progress_items`: 工事進行
- `push_subscriptions`: 通知購読端末

Row Level Securityにより会社単位でデータを分離します。

## 7. 環境変数

- `NEXT_PUBLIC_SUPABASE_URL`
- `NEXT_PUBLIC_SUPABASE_ANON_KEY`
- `NEXT_PUBLIC_VAPID_PUBLIC_KEY`
- `VAPID_PRIVATE_KEY`
- `VAPID_SUBJECT`

秘密鍵はVercelの環境変数で管理し、リポジトリへ保存しません。

## 8. セットアップ

新規環境ではSupabase SQL Editorで `supabase/setup.sql` を実行します。

既存環境へ通知だけ追加する場合は `supabase/RUN_PUSH_NOTIFICATIONS.sql` を実行します。

Supabase Auth推奨設定:

- Site URL: `https://kensapo.vercel.app`
- Redirect URL: `https://kensapo.vercel.app/auth/callback`
- 導入直後から利用する場合はメール確認を無効化

## 9. 非機能要件・制限

- HTTPS必須
- 日本語のみ
- オフライン利用非対応
- インターネット接続が必要
- iPhoneの通知はホーム画面追加後の利用を推奨
- バックアップは契約中のSupabaseプラン機能に依存
- 厳密なロール別画面制限は今後の拡張対象

## 10. UI方針

- 60代でも読みやすい文字サイズ
- 64px以上を基本とした主要タップ領域
- ホーム、予定、現場、設定の4メニュー
- 具体的な日本語ボタン
- 操作結果とエラーを日本語で表示
