## 1. システム概要

KenSapo は、建設現場における以下をスマホブラウザで一元管理する Web アプリケーションです。

- 現場（工事）情報の管理
- 作業員・行動予定の管理
- 現場写真の保存
- 日報の記録
- 日報の記録
- 工事進行チェックリスト

### 技術構成

| 層 | 技術 |
|----|------|
| フロント | Next.js 16 / TypeScript / Tailwind CSS |
| 認証・DB・Storage | Supabase |
| 日報 | （廃止） |
| ホスティング | Vercel |

## 2. ユーザーと権限

### 2.1 ロール

| DB値 | 表示名 | 想定される利用 |
|------|--------|----------------|
| admin | 管理者 | 会社設定・全機能 |
| manager | 現場監督 | 現場・予定・日報・写真 |
| worker | 作業員 | 予定の開始/終了・写真 |

※ 現時点では画面ごとの厳密な権限制御は最小限。同一会社内データは RLS で分離。

### 2.2 認証

- **メール＋パスワード**（Supabase Auth）
- **Passkey（顔認証・指紋）** … 設定画面から登録可能。ログイン画面には非表示（シンプル化のため）
- 新規登録時 `register_company` RPC で会社・プロフィールを自動作成

## 4. 機能仕様

### 4.1 現場（projects テーブル）

| 項目 | 仕様 |
|------|------|
| 必須 | 現場名 |
| 任意 | 住所、担当者、着工日、完了予定日、状態、メモ |
| 状態 | 未着手 / 進行中 / 完了 / 保留 |
| 登録後 | 標準工事進行チェック項目をバックグラウンドで自動投入 |

### 4.2 行動予定（schedules テーブル）

| 項目 | 仕様 |
|------|------|
| 関連 | 現場、作業員（任意）、日付 |
| 時刻 | 予定開始・終了、実績開始・終了 |
| 操作 | **作業を開始する** / **作業を終了する**（当日のみ） |
| 状態 | scheduled / in_progress / completed |

### 4.3 写真（site_photos + Storage）

- Supabase Storage に保存
- タイトル・コメント・撮影日を記録
- 署名 URL で表示

### 4.4 日報・日報（daily_reports）

| 項目 | 仕様 |
|------|------|
| 入力 | 日付、天気、人数、作業内容、資材、問題、翌日予定 |
| 保存後 | 日報日報一覧へ遷移 |
| （廃止） | `POST /api/generate-report`（認証必須、DBから日報取得） |
| 出力 | ブラウザ印刷 / PDF 用ページ |

### 4.5 工事進行チェック（project_progress_items）

- カテゴリ・工種・項目名でグループ化
- チェック ON/OFF で進捗管理
- 新規現場作成時に標準項目を自動作成

### 4.6 作業員（workers）

- 名前（必須）、電話、職種、状態（在籍/休止など）
- 予定への割り当てに使用

## 5. データベース

### 5.1 セットアップ

- **推奨:** `supabase/setup.sql` を SQL Editor で1回実行
- ポインタ: `supabase/RUN_THIS_IN_SUPABASE.sql`

### 5.2 主要テーブル

| 論理名 | 物理テーブル |
|--------|--------------|
| 会社 | companies |
| ユーザー情報 | profiles |
| 現場 | projects |
| 写真 | site_photos |
| 日報 | daily_reports（ai_report カラム） |
| 作業員 | workers |
| 予定 | schedules |
| 工事進行 | project_progress_items |

### 5.3 セキュリティ

- Row Level Security（会社単位でデータ分離）
- 認証 API は middleware + サーバーコンポーネントで検証

## 6. 環境変数（本番）

| 変数名 | 用途 |
|--------|------|
| NEXT_PUBLIC_SUPABASE_URL | Supabase 接続 URL |
| NEXT_PUBLIC_SUPABASE_ANON_KEY | 公開キー |
| （廃止） | 日報生成（任意） |

Vercel Project Settings で設定し、変更後は再デプロイが必要。

### Supabase 推奨設定

- Email confirmations: OFF（すぐ使う場合）
- Site URL: `https://kensapo.vercel.app`
- Redirect URLs: `https://kensapo.vercel.app/auth/callback`

## 7. UI・UX 方針（60代向け）

| 項目 | 仕様 |
|------|------|
| 基本文字サイズ | 20px |
| ボタン高さ | 64〜72px |
| 下部ナビ | ホーム・予定・現場・設定（アイコン付き） |
| ホーム | 大きな3ボタンを最上部に配置 |
| ログアウト | 設定画面のみ（誤タップ防止） |
| ローディング | 画面遷移時にスピナー表示 |
| エラー | 画面内に日本語で表示（alert 最小化） |

## 8. 非機能要件

| 項目 | 内容 |
|------|------|
| 対応ブラウザ | モダンモバイルブラウザ（Safari, Chrome） |
| オフライン | 非対応（要ネットワーク） |
| 多言語 | 日本語のみ |
| バックアップ | Supabase 側のバックアップに依存 |

## 9. 既知の制限（MVP）

- 作業員の編集・削除 UI は限定的
- 権限ごとの画面制限は未実装
- Passkey ログインは設定からの登録のみ（ログイン画面非表示）
- 日報は （廃止） 未設定時は利用不可
